# `smoovie` — HEALPix All-Sky Movie Command (L3) — Design

**Date:** 2026-06-15
**Status:** Approved (brainstorming) — ready for implementation plan
**Scope:** A hip-cargo CLI command `kremetart smoovie` that renders a sequence of TART HDF snapshots
into a HEALPix all-sky (Mollweide) movie via `mollview` + ffmpeg, plus a quantitative
steady-source pixel-stability test. This is the L3 deliverable deferred from the accuracy-
verification work (`docs/superpowers/specs/2026-06-15-accuracy-verification-design.md`).

---

## 1. Purpose

Show our pipeline working on real data: image each frame of a TART HDF sequence onto the fixed
equatorial HEALPix grid and animate them. The headline property is that the grid is **sidereally
fixed** — steady celestial sources hold a pixel while satellites drift — which is the input contract
of the Stage-2 detector. The accuracy comparison against tart2ms is **not** revisited here (it is
~arcmin, invisible on a ~180° all-sky plot, and already quantified in L1/L2); `smoovie` renders
**our** pipeline only.

## 2. Packaging (hip-cargo command)

Mirrors the existing `stream_msv4` command (`cli/` thin wrapper, `core/` implementation, generated
`cabs/` YAML):

- **`src/kremetart/cli/smoovie.py`** — `@stimela_cab(name="smoovie")` + `@stimela_output` Typer
  wrapper. Lazy-imports `core`, with the standard `--backend` / `--always-pull-images`
  `StimelaMeta(skip=True)` parameters and the native→container fallback (copy the `stream_msv4`
  shape exactly).
- **`src/kremetart/core/smoovie.py`** — the real implementation, no decorators; free to import
  matplotlib / healpy / ffmpeg-subprocess.
- **`src/kremetart/cabs/smoovie.yml`** — auto-generated by the pre-commit hook; never hand-edited.
- Registered in `src/kremetart/cli/__init__.py` next to `stream_msv4`.

## 3. CLI signature

```
kremetart smoovie --hdf-dir <dir> --movie <out.mp4> [--nside 128] [--fps 2] [--cmap inferno]
```

| Parameter | Type | Default | Notes |
|---|---|---|---|
| `hdf_dir` | `StimelaMeta(type="Directory")` | required | globs `*.hdf`, sorted by name (timestamps sort chronologically) |
| `movie` | `@stimela_output(dtype="File")` | required | output `.mp4` |
| `nside` | int | 128 | HEALPix resolution (~0.46° pixels) |
| `fps` | int | 2 | frames per second |
| `cmap` | str | `"inferno"` | matplotlib colormap |

Plus `backend` and `always_pull_images` (`StimelaMeta(skip=True)`), exactly as in `stream_msv4`.

## 4. Core data flow (`core/smoovie.py`)

1. Collect `sorted(hdf_dir.glob("*.hdf"))`; error clearly if empty.
2. Per file: `read_hdf_as_msv4` → extract the ITRS baselines from the partition's `antenna_xds` →
   `image_frame(vis, weights, times, itrs_baselines, pix_vec, freqs, xp=np)` → `(npix,)` dirty map.
   Weights are unit; the **full sphere** is rendered (no below-horizon masking).
3. Compute a **single** `vmin/vmax` from percentiles over **all** maps (fixed scale across frames →
   no autoscale flicker; brightness comparable frame-to-frame).
4. Per frame: `healpy.mollview(map, title=<UTC timestamp>, cmap=cmap, min=vmin, max=vmax,
   nest=True, graticule on)` → save PNG into a `tempfile.TemporaryDirectory()`.
5. Encode: `ffmpeg` subprocess turns the PNG sequence into `movie` at `fps`. Return the path.

**Shared baseline helper.** `image_frame` needs ITRS baseline vectors `(nbl, 3)`. The logic already
exists privately as `kremetart.utils.rephasing._itrs_baselines`. Promote it to a public shared
helper (e.g. `kremetart.utils.rephasing.itrs_baselines`, keeping a thin private alias for existing
callers) so `core/smoovie.py` and the rephasing module share one implementation rather than
duplicating it.

**Frequencies / pol.** Single channel (`frequency` from the datatree), scalar single-pol — same
contract `image_frame` already expects.

## 5. Verification — steady-source pixel stability

`tests/test_smoovie_stability.py`: simulate one **fixed-ICRS** source and image it at each of the 9
HDFs' representative timestamps (reusing `tests.accuracy_helpers.simulate_visibilities`,
`image_frame`, and `recovered_direction_and_flux`). **Assert the recovered peak pixel index is
identical across all frames** — the quantitative proof that the equatorial grid is sidereally fixed
(a steady source does not drift), which is the Stage-2 detector's input contract. Uses the
hemisphere-constrained search (`near=`) to avoid the coplanar mirror. Deterministic, CPU.

## 6. Testing

- **Round-trip (mandatory):** add a `smoovie` case to `tests/test_roundtrip.py`
  (cli→cab→cli byte-identical after `ruff format`). New CLI commands must round-trip.
- **Stability test:** §5.
- **Core smoke test:** `tests/test_smoovie_core.py` runs `core.smoovie` on the bundled 9 HDFs to a
  temp `.mp4` and asserts the file exists and is non-empty. Skips if ffmpeg is absent
  (`shutil.which("ffmpeg") is None`) or matplotlib/healpy unavailable.

## 7. Dependencies

- Add **`matplotlib`** to `[project.optional-dependencies].full` (healpy already a core dep;
  numpy/astropy already in `full`). Imported only from `core/`.
- **ffmpeg**: a system binary (present in dev and in the container image). `core/smoovie.py` raises a
  clear `RuntimeError` if `shutil.which("ffmpeg")` is `None`; the smoke test skips in that case.

## 8. Out of scope

- Side-by-side vs tart2ms / disko (settled in L1/L2; not evidentiary in a movie).
- Below-horizon masking (full sphere chosen) and primary-beam weighting.
- GPU rendering — the batch tool runs `image_frame` with `xp=np`; a `cupy` path can be added later.
- Per-frame source/satellite annotation overlays.
